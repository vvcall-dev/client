# P2P Voice

Rust で構築されたピアツーピア音声通信アプリケーションで、リアルタイムオーディオストリーミング、自動 NAT トラバーサル、モダンな GUI を備えています。

## 機能

- **ピアツーピアアーキテクチャ**: 中央メディアサーバーなしでクライアント間の直接 UDP 通信
- **自動 NAT トラバーサル**: シームレスな接続のための組み込み STUN とホールパンチング
- **リアルタイムオーディオ**: 48kHz の Opus コーデックを使用した低遅延音声チャット
- **アダプティブジッターバッファ**: スムーズなオーディオ再生のための動的バッファ調整
- **音声活動検出**: ハングオーバー防止機能を備えた音声活動に基づく自動送信
- **モダンな GUI**: egui/eframe で構築されたクリーンなインターフェース
- **クロスプラットフォーム**: Windows、Linux、macOS（Intel および Apple Silicon）をサポート
- **自動更新**: 組み込みの更新チェッカーおよびダウンローダー
- **永続的な設定**: セッション間での設定の保存
- **ミュート/ディーフェンコントロール**: プライバシーコントロールへのクイックアクセス
- **ピングモニタリング**: 各ピアのリアルタイムレイテンシ表示

## テクノロジースタック

- **言語**: Rust 2024 Edition
- **GUI フレームワーク**: Glow レンダラーを使用する eframe/egui
- **オーディオバックエンド**: cpal (Cross-Platform Audio Library)
- **オーディオコーデック**: Opus（VoIP 向けに最適化）
- **ネットワーキング**: 
  - シグナリングとピアディスカバリのための WebSocket
  - ピアツーピアオーディオストリーミングのための直接 UDP
- **TLS**: 安全な接続のための rustls

## インストール

### リリースから

[リリースページ](https://github.com/tallfly/p2p-voice/releases) からお使いのプラットフォーム向けの最新リリースをダウンロードしてください：

### ソースからのビルド

#### 前提条件

**全プラットフォーム:**
- Rust（安定版ツールチェーン）
- CMake

**Linux (Ubuntu/Debian):**
```bash
sudo apt-get install libasound2-dev pkg-config cmake build-essential libssl-dev
```

**Windows:**
- C++ サポート付き Visual Studio Build Tools

**macOS:**
- Xcode Command Line Tools

#### ビルド手順

```bash
# リポジトリをクローン
git clone https://github.com/tallfly/p2p-voice.git
cd p2p-voice

# リリースモードでビルド
cargo build --release
```

バイナリは以下の場所に配置されます：
- **Windows**: `target/release/p2p-voice.exe`
- **Linux/macOS**: `target/release/p2p-voice`

## 使用方法

1. **アプリケーションを起動**
2. **ログイン/登録**: サーバーで認証するためのユーザー名とパスワードを入力
3. **ルームに参加**: 他のユーザーと接続するためのルーム名を入力
4. **オーディオデバイス**: 入力（マイク）と出力（スピーカー）デバイスを選択
5. **通信**: 話し始めてください！アプリは音声活動を自動的に検出し、オーディオを送信します

### コントロール

- **ミュート**: マイク送信を無効化
- **ディーフェン**: オーディオ出力を無効化（他のユーザーの音が聞こえない）
- **ボリュームコントロール**: 個別のピアの音量を調整
- **オーバーレイモード**: オプションのオーバーレイ表示

### 設定

設定は自動的に保存され、永続化されます：
- サーバー URL（デフォルト：`p2p.tallfly.me`）
- ユーザー名と認証トークン
- 選択されたオーディオデバイス
- オーバーレイの設定

## アーキテクチャ

### 音声エンジン

音声エンジンは以下を処理します：
- **オーディオキャプチャ**: ハードウェアサンプルレートでマイク入力をキャプチャ
- **リサンプリング**: Opus エンコードのために 48kHz に変換
- **エンコーディング**: Opus コーデックを使用してオーディオを圧縮（VoIP 最適化）
- **送信**: エンコードされたパケットを UDP で全てのピアに送信
- **受信**: シーケンス番号付きでピアからパケットを受信
- **ジッターバッファ**: ネットワークの変動を処理するためのアダプティブバッファリング
- **デコーディング**: Opus パケットを PCM オーディオに解凍
- **ミキシング**: 複数のピアからのオーディオを結合
- **再生**: ミキシングされたオーディオをスピーカーに出力

### ネットワーキング

1. **シグナリング**: ピアディスカバリのためのサーバーへの WebSocket 接続
2. **NAT トラバーサル**: 
   - パブリック IP を発見するための STUN クエリ
   - 直接接続を確立するためのホールパンチングパケット
3. **直接通信**: 低遅延オーディオストリーミングのための UDP ソケット
4. **キープアライブ**: レイテンシモニタリングと接続ヘルスのための Ping/Pong メカニズム

### パケット形式

- **オーディオパケット**: `[seq_num(2 バイト)][opus_payload]`
- **ホールパンチ**: `HOLE_PUNCH`（10 バイト）
- **Ping**: `PING[timestamp(8 バイト)]`（12 バイト）
- **Pong**: `PONG[timestamp(8 バイト)]`（12 バイト）

## 開発

### 開発モードでの実行

```bash
cargo run
```

### リリースビルドの最適化

リリースプロファイルは最大限の最適化で構成されています：
- LTO（リンクタイム最適化）を有効化
- より良い最適化のための単一コード生成ユニット
- より小さなバイナリのための panic abort
