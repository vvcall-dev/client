# P2P Voice

Приложение для голосовой связи peer-to-peer, созданное на Rust, с потоковой передачей аудио в реальном времени, автоматической обходом NAT и современным графическим интерфейсом.

## Возможности

- **Архитектура Peer-to-Peer**: Прямая UDP-связь между клиентами без центрального медиа-сервера
- **Автоматический обход NAT**: Встроенные STUN и hole-punching для беспрепятственного подключения
- **Аудио в реальном времени**: Голосовой чат с низкой задержкой с использованием кодека Opus при 48 кГц
- **Адаптивный джиттер-буфер**: Динамическая настройка буфера для плавного воспроизведения аудио
- **Обнаружение голосовой активности**: Автоматическая передача на основе голосовой активности с защитой от обрывов
- **Современный GUI**: Чистый интерфейс, созданный с помощью egui/eframe
- **Кроссплатформенность**: Поддержка Windows, Linux и macOS (Intel и Apple Silicon)
- **Автообновление**: Встроенная проверка обновлений и загрузчик
- **Постоянные настройки**: Сохранение конфигурации между сеансами
- **Управление mute/deafen**: Быстрый доступ к элементам управления конфиденциальностью
- **Мониторинг пинга**: Отображение задержки в реальном времени для каждого пира

## Технологический стек

- **Язык**: Rust 2024 Edition
- **GUI фреймворк**: eframe/egui с рендерером Glow
- **Аудио бэкенд**: cpal (Cross-Platform Audio Library)
- **Аудио кодек**: Opus (оптимизирован для VoIP)
- **Сеть**: 
  - WebSocket для сигнализации и обнаружения пиров
  - UDP для прямой потоковой передачи аудио между пирами
- **TLS**: rustls для безопасных соединений

## Использование

1. **Запустите приложение**
2. **Вход/Регистрация**: Введите имя пользователя и пароль для аутентификации на сервере
3. **Присоединиться к комнате**: Введите название комнаты для подключения к другим пользователям
4. **Аудиоустройства**: Выберите предпочтительные устройства ввода (микрофон) и вывода (динамики)
5. **Общение**: Начните говорить! Приложение автоматически обнаруживает голосовую активность и передает аудио

### Управление

- **Mute**: Отключить передачу микрофона
- **Deafen**: Отключить аудиовыход (не слышно других)
- **Регулировка громкости**: Настройка громкости отдельных пиров
- **Режим наложения**: Опциональное отображение наложения

### Конфигурация

Настройки автоматически сохраняются и сохраняются между сеансами:
- URL сервера (по умолчанию: `p2p.tallfly.me`)
- Имя пользователя и токен аутентификации
- Выбранные аудиоустройства
- Настройки наложения

## Архитектура

### Голосовой движок

Голосовой движок обрабатывает:
- **Захват аудио**: Захват входного сигнала микрофона с частотой дискретизации оборудования
- **Передискретизация**: Преобразование в 48 кГц для кодирования Opus
- **Кодирование**: Сжатие аудио с использованием кодека Opus (оптимизировано для VoIP)
- **Передача**: Отправка закодированных пакетов через UDP всем пирам
- **Прием**: Получение пакетов от пиров с номерами последовательностей
- **Джиттер-буфер**: Адаптивная буферизация для обработки вариаций сети
- **Декодирование**: Декомпрессия пакетов Opus обратно в PCM аудио
- **Микширование**: Комбинирование аудио от нескольких пиров
- **Воспроизведение**: Вывод микшированного аудио на динамики

### Сеть

1. **Сигнализация**: WebSocket соединение с сервером для обнаружения пиров
2. **Обход NAT**: 
   - STUN запрос для обнаружения публичного IP
   - Hole-punching пакеты для установления прямых соединений
3. **Прямая связь**: UDP сокеты для потоковой передачи аудио с низкой задержкой
4. **Keep-alive**: Механизм Ping/Pong для мониторинга задержки и состояния соединения

### Формат пакетов

- **Аудио пакеты**: `[seq_num(2 байта)][opus_payload]`
- **Hole Punch**: `HOLE_PUNCH` (10 байт)
- **Ping**: `PING[timestamp(8 байт)]` (12 байт)
- **Pong**: `PONG[timestamp(8 байт)]` (12 байт)

### Оптимизация сборки релиза

Профиль релиза настроен с максимальной оптимизацией:
- LTO (Link Time Optimization) включена
- Единый блок кодогенерации для лучшей оптимизации
- Panic abort для уменьшения размера бинарных файлов
